
## Implementing DQNet's Dynamic Quantization on ResNet-18 for CIFAR-10

### Phase 1: Set up your quantization primitives

**Step 1 — Implement a quantize function.**
You need a differentiable quantizer that rounds activations/weights to N bits. Use the Straight-Through Estimator (STE) for backprop — forward pass rounds, backward pass passes gradient through unchanged:

```python
class STEQuantize(torch.autograd.Function):
    @staticmethod
    def forward(ctx, x, bits):
        scale = (2**bits - 1)
        return torch.round(x * scale) / scale

    @staticmethod
    def backward(ctx, grad):
        return grad, None  # STE: pass gradient through unchanged
```

**Step 2 — Pick your candidate bit-widths.**
DQNet uses a discrete set, e.g. `{2, 4, 8}` bits. Every layer will be assigned one of these per input at runtime.

---

### Phase 2: Build the bit-controller

A lightweight bit-controller is established and trained jointly with the given neural network. During inference, the quantization configuration for an arbitrary image is determined by the bit-widths generated by the controller — an image with simple texture will be allocated with lower bits and computational complexity, and vice versa.

**Step 3 — Design the controller architecture.**
The controller is a small CNN that takes the raw input image and outputs one bit-width decision per layer. The paper ablates the number of shared convolutional layers and finds that using only one layer performs worse than static quantization, because the expression ability of the bit-controller is not powerful enough. Performance improves as the number of layers increases. Use 2–3 conv layers as a safe starting point:

```python
class BitController(nn.Module):
    def __init__(self, num_layers, num_bits_options=3):
        # num_layers = number of layers in ResNet-18 you want to control
        super().__init__()
        self.shared = nn.Sequential(
            nn.Conv2d(3, 16, 3, stride=2, padding=1),  # 32->16
            nn.ReLU(),
            nn.Conv2d(16, 32, 3, stride=2, padding=1), # 16->8
            nn.ReLU(),
            nn.AdaptiveAvgPool2d(1),
            nn.Flatten()
        )
        # One head per layer, outputting logits over bit choices
        self.heads = nn.ModuleList([
            nn.Linear(32, num_bits_options) for _ in range(num_layers)
        ])

    def forward(self, x):
        feat = self.shared(x)
        # Returns list of logits, one per layer
        return [head(feat) for head in self.heads]
```

**Step 4 — Add Gumbel-Softmax for differentiable discrete selection during training.**
You need to pick a discrete bit-width, but still backprop through the controller:

```python
def select_bits(logits, bit_options, temperature=1.0, hard=True):
    # During training: differentiable soft sample
    soft = F.gumbel_softmax(logits, tau=temperature, hard=hard)
    # soft is one-hot over bit_options
    bits = (soft * torch.tensor(bit_options, dtype=torch.float)).sum()
    return bits
```
At inference, replace Gumbel-Softmax with a plain `argmax`.

---

### Phase 3: Modify ResNet-18 to accept dynamic bit-widths

**Step 5 — Wrap each layer's forward pass to accept a `bits` argument.**
The backbone itself doesn't change structurally — you just intercept the forward pass and apply quantization with the controller-selected bit-width before each conv:

```python
class DynamicQuantConv2d(nn.Conv2d):
    def forward(self, x, bits):
        w_q = STEQuantize.apply(self.weight, bits)
        x_q = STEQuantize.apply(x, bits)
        return F.conv2d(x_q, w_q, self.bias, self.stride, self.padding)
```

Replace all `nn.Conv2d` layers in ResNet-18 with `DynamicQuantConv2d`.

**Step 6 — Modify ResNet-18's forward() to accept and route bit-width decisions.**
The controller produces a list of bit decisions (one per layer). Pass them through:

```python
def forward(self, x):
    bit_logits = self.controller(x)
    bits_per_layer = [select_bits(l, self.bit_options) for l in bit_logits]
    
    out = self.conv1(x, bits_per_layer[0])
    # ... pass bits_per_layer[i] to each subsequent layer
    return out
```

---

### Phase 4: Training

**Step 7 — Define the loss.**
You have two competing objectives: accuracy and BitOPs. Combine them:

```python
loss = CE_loss(logits, labels) + lambda * mean_bitops(bits_per_layer, layer_sizes)
```

where `mean_bitops` computes the weighted average BitOPs given the selected bit-widths and the number of operations in each layer. `lambda` is a hyperparameter that controls the accuracy/efficiency tradeoff — sweep it to generate your tradeoff curve.

**Step 8 — Train jointly from scratch.**
Train the backbone and controller together end-to-end. Standard SGD with momentum, cosine LR schedule, ~200 epochs on CIFAR-10. The controller learns which input features correlate with "difficulty" without any explicit supervision — it only sees the combined loss signal.

---

### Phase 5: Evaluation

**Step 9 — At inference, swap Gumbel-Softmax for argmax.**
```python
bits = bit_options[torch.argmax(logits).item()]
```

**Step 10 — Report average BitOPs across the test set**, not just accuracy. For each test image, log which bit-width was selected per layer, compute BitOPs for that configuration, and average. This is your primary efficiency metric. Wall-clock latency is secondary and noisy at this scale.

---

### Key hyperparameters to tune
- Controller depth: 2–3 shared conv layers (per paper ablation)
- Bit options: `{2, 4, 8}` or `{3, 4, 5}` depending on target regime
- Lambda (BitOPs penalty weight): sweep to get the accuracy-efficiency tradeoff curve
- Gumbel temperature: start at 1.0, optionally anneal down during training
